# Local Dev & E2E Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up a fully local development environment (Supabase CLI + Market dev server) with isolated E2E testing for the CLI, so `aas install/enable/sync` can be verified end-to-end without affecting the developer's real `~/.claude` or `~/.codex` configs.

**Architecture:** Supabase CLI manages a local Postgres + Auth + PostgREST stack in Docker; the Next.js market runs with `pnpm dev` pointing at localhost:54321; the CLI binary runs on the host with `AAS_HOME`, `CLAUDE_CONFIG_DIR`, and `CODEX_CONFIG_DIR` redirected to `/tmp` dirs, completely isolating test writes from real configs. A `Makefile` at the repo root wraps all operations.

**Tech Stack:** Supabase CLI (brew), Docker, Next.js 14, Bun (CLI binary), GNU Make, Bash

## Global Constraints

- Supabase project dir: `supabase/` at repo root (not inside `apps/market/`)
- Migration: `supabase/migrations/001_initial.sql` — moved from `apps/market/supabase/migrations/`
- Seed: `supabase/seed.sql` — applied automatically by `supabase db reset` after migrations
- E2E isolation: `AAS_HOME=/tmp/aas-e2e`, `CLAUDE_CONFIG_DIR=/tmp/claude-e2e`, `CODEX_CONFIG_DIR=/tmp/codex-e2e`
- E2E script cleans up `/tmp/aas-e2e`, `/tmp/claude-e2e`, `/tmp/codex-e2e` at start and end
- Root `.gitignore` already covers `.env.local` — no changes needed to gitignore
- CLI binary already defaults to `baseUrl = http://localhost:3000` — no code changes to CLI
- `AAS_HOME`, `CLAUDE_CONFIG_DIR`, `CODEX_CONFIG_DIR` are read from `process.env` in `apps/client-core/src/paths.ts` — no code changes needed
- Makefile targets: `setup`, `seed`, `dev`, `build-cli`, `e2e`, `stop`, `status`
- `make setup` is one-time only; `make dev` and `make e2e` are used daily

- [ ] **Step 2: Create `scripts/local-e2e.sh`**

Write `scripts/local-e2e.sh`:

```bash
#!/usr/bin/env bash
# E2E test for the full CLI → SDK → Market → Supabase chain.
# Runs in isolated /tmp dirs — never touches ~/.claude or ~/.codex.
set -euo pipefail

export AAS_HOME=/tmp/aas-e2e
export CLAUDE_CONFIG_DIR=/tmp/claude-e2e
export CODEX_CONFIG_DIR=/tmp/codex-e2e

AAS="$(cd "$(dirname "$0")/.." && pwd)/bin/aas"

echo "=== [setup] cleaning isolated dirs ==="
rm -rf "$AAS_HOME" "$CLAUDE_CONFIG_DIR" "$CODEX_CONFIG_DIR"
mkdir -p "$AAS_HOME" "$CLAUDE_CONFIG_DIR/skills" "$CODEX_CONFIG_DIR"

echo ""
echo "=== search: verify market is reachable ==="
$AAS search test

echo ""
echo "=== install: all 3 test items ==="
$AAS install openai-provider-test
$AAS install hello-skill
$AAS install fs-mcp-test

echo ""
echo "=== list: all items installed and enabled ==="
$AAS list

echo ""
echo "=== disable + re-enable: test both commands ==="
$AAS disable openai-provider-test --for claude
$AAS list
$AAS enable openai-provider-test --for claude
$AAS list

echo ""
echo "=== sync: idempotency check ==="
$AAS sync

echo ""
echo "=== info: spot-check one item ==="
$AAS info openai-provider-test

echo ""
echo "=== verify: checking config files ==="

grep -q '"openai-provider-test"' "$CLAUDE_CONFIG_DIR/settings.json" \
  && echo "✓ provider in claude settings" \
  || { echo "✗ provider missing from claude settings"; exit 1; }

grep -q '"fs-mcp-test"' "$CLAUDE_CONFIG_DIR/settings.json" \
  && echo "✓ mcp in claude settings" \
  || { echo "✗ mcp missing from claude settings"; exit 1; }

[ -s "$CLAUDE_CONFIG_DIR/skills/hello-skill.md" ] \
  && echo "✓ hello-skill.md exists and non-empty" \
  || { echo "✗ hello-skill.md missing or empty"; exit 1; }

grep -q '"openai-provider-test"' "$CODEX_CONFIG_DIR/config.yaml" 2>/dev/null \
  && echo "✓ provider in codex config" \
  || echo "⚠ codex config not checked (codex may not be installed)"

echo ""
echo "=== [cleanup] removing isolated dirs ==="
rm -rf "$AAS_HOME" "$CLAUDE_CONFIG_DIR" "$CODEX_CONFIG_DIR"

echo ""
echo "✓ All E2E checks passed"
```

## Task 4: Makefile

**Files:**

**Interfaces:**
- Consumes: all prior tasks; `supabase` CLI on PATH; `pnpm` on PATH; `bin/aas` binary
- Produces: 7 Make targets that wrap the full local dev workflow

- [ ] **Step 1: Write `Makefile`**

```makefile
# Makefile — Local development workflow for ai-agent-store
# Requires: supabase CLI, Docker, pnpm, Bun

.PHONY: setup seed dev build-cli e2e stop status

## One-time setup: install CLI, init, start Supabase, seed data, create .env.local
setup:
	brew install supabase/tap/supabase
	supabase init
	supabase start
	$(MAKE) seed
	@echo ""
	@echo "Next: create apps/market/.env.local with credentials from 'make status'"
	@echo "Then: make dev"

## Re-apply migrations + seed (resets all local data)
seed:
	supabase db reset

## Start local dev: Supabase (if not running) + market on :3000
dev:
	supabase start
	pnpm --filter=@aas/market dev

## Compile CLI binary to bin/aas
build-cli:
	pnpm --filter=@aas/cli build:bin

## Run full E2E test in isolated /tmp dirs (builds CLI first)
e2e: build-cli
	@./scripts/local-e2e.sh

## Stop Supabase local stack
stop:
	supabase stop

## Print local Supabase credentials (URL, anon key, service_role key)
status:
	supabase status
```
